IFS

IFS is de Internal Field Separator. Daarmee bepaalt bash op welke tekens input opgesplitst wordt in velden of woorden. Standaard zijn dat spatie, tab en newline. ŅłĆfileciteŅłéturn2file2ŅłéL3-L3Ņłü

IFS='
'
for state in $(cat states.txt)
do
  echo $state
done

De les toont het probleem met New York, dat zonder aangepaste IFS in twee woorden opgesplitst wordt. ŅłĆfileciteŅłéturn2file2ŅłéL4-L7Ņłü

getopts

while getopts ':ab:c' opt
do
  case "$opt" in
    a) echo "Found the -a option" ;;
    b) echo "Found the -b option, with value $OPTARG" ;;
    c) echo "Found the -c option" ;;
    *) echo "Unknown option: $opt" ;;
  esac
done

getopts verwerkt opties zoals -a, -b test1 en -c. In de cursus wordt ook getoond dat b: betekent dat optie -b een argument verwacht. ŅłĆfileciteŅłéturn2file2ŅłéL8-L9Ņłü

Functions

function func1 {
  echo "This is an example of a function"
}

count=1
while [ $count -le 5 ]
do
  func1
  count=$((count + 1))
done

Exit status

De standaard exitstatus van een functie is de exitstatus van het laatste commando in die functie. Met return kan je die aanpassen. Je leest die direct uit met $?. ŅłĆfileciteŅłéturn2file2ŅłéL17-L19Ņłü

func1() {
  ls -l badfile
}
func1
echo $?

Function output

dbl() {
  read -p "Enter a value: " value
  echo $((value * 2))
}
result=$(dbl)
echo "The new value is $result"

Function parameters

adden() {
  if [ $# -eq 0 ] || [ $# -gt 2 ]; then
    echo -1
  elif [ $# -eq 1 ]; then
    echo $(( $1 + $1 ))
  else
    echo $(( $1 + $2 ))
  fi
}
