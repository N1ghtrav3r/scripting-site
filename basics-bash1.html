Bash

Bash is een shell en ook de interpreter die je script uitvoert.

Script

Een script is gewoon een tekstbestand met commando’s die je normaal ook manueel zou typen.

Belangrijk

Let op met uitvoerrechten, de shebang, en het verschil tussen variabele instellen en uitlezen.

Shells

Op één systeem kunnen meerdere shells aanwezig zijn. In Linux is bash heel populair. Op macOS is zsh vaak standaard, en op Windows heb je onder meer Command Prompt en PowerShell.

Standaard login shell

De standaard shell van een gebruiker staat in /etc/passwd.

root:x:0:0:root:/root:/bin/bash
jef:x:1001:1001::/home/jef:/bin/dash

Shell wijzigen

chsh

Daarmee kan je de login shell aanpassen.

Wat is een bash script?

  • Een bash script is een platte tekstbestand.
  • Daarin staat een reeks commando’s.
  • Die commando’s kan je vaak ook gewoon rechtstreeks in de terminal typen.
  • Een script moet niet gecompileerd worden.
  • De extensie .sh is gebruikelijk, maar niet verplicht.

Belangrijk principe

Wat je op de commandline kan uitvoeren, kan meestal ook in een script. En omgekeerd.

Hoe werkt dat intern?

Wanneer jij in de terminal werkt, draait er al een bash-proces. Start je een script, dan wordt dat typisch uitgevoerd in een nieuw proces. Je script draait dus niet gewoon “in dezelfde regel verder”.

Script uitvoeren

Voor je een script rechtstreeks kan uitvoeren, moet het uitvoerbaar zijn.

Uitvoerrecht geven

chmod +x myscript.sh

Script starten

./myscript.sh

Als je execute permission vergeet

Dan krijg je meestal een fout zoals Permission denied.

Bash rechtstreeks gebruiken

bash myscript.sh
bash -x myscript.sh

-x is handig om te debuggen, want dan toont bash stap voor stap wat uitgevoerd wordt.

Shebang

De shebang is de eerste regel van je script. Daarmee zeg je welke interpreter het script moet uitvoeren.

#!/bin/bash
# Een commentaar

echo "Hallo studenten"

Regels die je moet onthouden

  • De shebang moet op de allereerste regel staan.
  • Geen spaties vóór #!.
  • Geen spaties tussen ! en het pad.
  • Gebruik liefst een absoluut pad zoals /bin/bash.

Zonder shebang?

Dat kan soms werken als je het script vanuit bash start, maar het is geen goede gewoonte. Je maakt je script daar minder betrouwbaar mee.

Andere voorbeelden

#!/usr/bin/python3
#!/usr/bin/perl

file command

Met file laat je Linux bepalen wat voor soort bestand iets is.

file script
file /bin/bash

Wat test file ongeveer?

  • Filesystem tests: directory, symlink, device, …
  • Magic tests: interne bytes van het bestand
  • Language tests: Bash, Perl, Python, …

De eerste test die matcht, bepaalt het resultaat.

Variabelen

Met variabelen doe je in bash vooral twee dingen:

  • een waarde opslaan
  • een waarde later terug uitlezen

Belangrijk verschil

  • Bij instellen gebruik je geen $
  • Bij uitlezen gebruik je wel $
naam="Gunther"
echo "$naam"

Waarom die $?

Voor bash betekent $naam: vervang deze variabele door haar inhoud vóór de regel uitgevoerd wordt.

Case sensitive

naam="Gunther"
Naam="Bram"
echo "$naam"
echo "$Naam"

Dat zijn twee verschillende variabelen.

Command line arguments

Als je een script uitvoert met extra waarden erachter, dan zijn dat command line arguments.

./script.sh hallo wereld

In je script kan je die dan aanspreken met:

echo "$1"
echo "$2"

Voorbeeld

#!/bin/bash
echo "Eerste argument: $1"
echo "Tweede argument: $2"

Het systeem vult die automatisch in wanneer je het script start.

Eigen variabelen instellen

De basisvorm is:

variable=value

Heel belangrijk

Er mogen geen spaties rond het = teken staan.

myvariable="hello"
anothervar="Fred"
echo "$myvariable $anothervar"

echo

sampledir="/etc"
ls "$sampledir"

Onthouden

  • Gebruik beschrijvende namen
  • Gebruik quotes als er kans is op spaties
  • Wees consequent met hoofdletters en kleine letters
