Wat is PowerShell?

PowerShell is een shell en scriptingtaal van Microsoft. In deze les ligt de nadruk op het beheren van systemen via commando’s en scripts.

  • Cmdlet = basiscommando
  • Function = eigen uitbreiding
  • Script = meerdere opdrachten samen
  • Module = bundel functionaliteit

In tegenstelling tot klassieke shells werkt PowerShell sterk met gestructureerde output en objecten, maar in les 4 ligt de focus nog vooral op de basis en het help-systeem.

Waarom PowerShell?

  • Veel Microsoft-producten werken er al mee.
  • Windows Server kan er volledig mee beheerd worden.
  • Het is de standaard manier om veel beheertaken te automatiseren.

Kleine historiek

  • 2005: PowerShell 1.0
  • 2009: PowerShell 2.0
  • 2012: PowerShell 3.0
  • 2013: PowerShell 4.0
  • 2016: PowerShell 5.0 en 5.1
  • Daarna: PowerShell Core / open source

Verb-Noun naamgeving

PowerShell-cmdlets volgen bijna altijd de vorm:

Werkwoord-Naamwoord

Voorbeelden

Get-Help
Update-Help
Get-Command
Group-Object
Sort-Object
Get-Process
Stop-Process

Belangrijk

  • Het tweede deel is normaal enkelvoudig
  • Dus Get-Service, niet Get-Services

Die naamgevingsregel is één van de kernpunten uit de slides. fileciteturn0file4L13-L15

Aliassen

PowerShell is niet hoofdlettergevoelig en heeft veel aliassen. Dat maakt de start makkelijker, maar je moet wel weten wat er achter zo’n alias zit.

Voorbeeld

dir
ls
Get-ChildItem
gci

Die kunnen allemaal naar hetzelfde verwijzen.

Alias controleren

Get-Alias gci
Get-Alias -Definition Get-ChildItem

Onthouden

  • dir, ls en gci zijn handige snelkoppelingen
  • In scripts is de volledige cmdletnaam vaak duidelijker

De slides tonen expliciet het voorbeeld met Get-ChildItem en de aliassen dir, gci en ls. fileciteturn0file4L19-L19

Help systeem

In deze les is de boodschap heel duidelijk: Help is your friend. Je moet niet alles van buiten leren als je vlot hulp kunt opzoeken.

Basis

Get-Help Get-Service

Niveaus van help

Get-Help Get-Service -Detailed
Get-Help Get-Service -Full
Get-Help Get-Service -Examples
Get-Help Get-Service -Online

Andere manieren

Help Get-Service
Man Get-Service
Get-Service -?

Updatable help

Update-Help

Dat kan nodig zijn omdat niet altijd alle helpbestanden standaard aanwezig zijn. In offline situaties kan je ook met Save-Help en Update-Help -SourcePath werken. fileciteturn0file4L26-L30

Handige tip

Get-Help -ShowWindow Get-Service

Zo krijg je help in een apart venster.

Help zoeken

Als je de exacte naam van een commando niet weet, kan je zoeken met wildcards of met Get-Command.

Zoeken in help

Get-Help *help*

Zoeken naar commando’s

Get-Command *help*

Werkwoorden bekijken

Get-Command -Verb *
Get-Command -Verb * | Group-Object Verb
Get-Command -Verb * | Group-Object Verb -NoElement | Sort-Object Count -Descending

Dit deel komt letterlijk terug in de les: zoeken, combineren, groeperen en sorteren. fileciteturn0file4L14-L15

Parameters

PowerShell-cmdlets hebben parameters. In de slides worden deze soorten benadrukt:

  • verplicht
  • optioneel
  • positioneel
  • switch

Belangrijke regel

Een parameternaam begint altijd met een -.

Voorbeelden

Get-Help Get-Service -Full
Get-Help Get-Service -Examples
Get-Command -Verb Get

Switch parameter

Een switch is een parameter zonder extra waarde. Bijvoorbeeld:

Get-Help Get-Service -Online

De les bespreekt parameters expliciet via Get-Help Get-Service en de verschillende parametercategorieën. fileciteturn0file4L33-L35

Eerste stappen

In de slides staat een eerste eenvoudig voorbeeld van systeembeheer via PowerShell:

Rename-Computer -NewName "psBram"
Restart-Computer
$env:COMPUTERNAME

Wat doet dit?

  • Rename-Computer wijzigt de computernaam
  • Restart-Computer herstart de machine
  • $env:COMPUTERNAME toont de naam uit de omgevingsvariabelen

Niet zomaar op een productiesysteem testen natuurlijk.
