IFS

IFS betekent Internal Field Separator. Dat is de variabele die bepaalt op welke tekens bash invoer opsplitst in woorden of velden.

Standaard splitst bash op

  • spatie
  • tab
  • newline

Waarom is dat belangrijk?

Als bash een lijst verwerkt, kan een waarde met een spatie erin per ongeluk in twee stukken gesplitst worden.

for item in $(cat states.txt)
do
  echo $item
done

Als in dat bestand New York staat, dan krijg je vaak New en York apart.

IFS voorbeelden

Alleen op newline splitsen

IFS='\n'
for state in $(cat states.txt)
do
  echo $state
done

Dat helpt als je regels met spaties wilt samenhouden.

Nette aanpak met oude IFS bewaren

OLDIFS=$IFS
IFS=$'\n'

for state in $(cat states.txt)
do
  echo "$state"
done

IFS=$OLDIFS

/etc/passwd splitsen

#!/bin/bash

IFSOLD=$IFS
IFS=$'\n'

for entry in $(cat /etc/passwd)
do
  echo "values in $entry"
  IFS=':'
  for ent in $entry
  do
    echo "deel = $ent"
  done
done

IFS=$IFSOLD

Belangrijk: als je IFS aanpast, zet hem daarna best terug. Anders kan latere code onverwacht gedrag geven.

getopts

Met getopts verwerk je opties die je meegeeft aan je script, zoals -a, -b test1 of -c.

Basisvoorbeeld

#!/bin/bash

while getopts ':ab:c' opt
do
  case "$opt" in
    a) echo "Found the -a option" ;;
    b) echo "Found the -b option, with value $OPTARG" ;;
    c) echo "Found the -c option" ;;
    *) echo "Unknown option: $opt" ;;
  esac
done

Hoe lees je ':ab:c'?

  • a = optie zonder argument
  • b: = optie die een argument verwacht
  • c = optie zonder argument

Voorbeeld van aanroep

./test19.sh -ab test1 -c

Belangrijk

  • $opt bevat de gevonden optie
  • $OPTARG bevat het argument van een optie die er één verwacht
  • getopts is veel properder dan zelf met $1, $2, ... te prutsen

BASH_SOURCE

BASH_SOURCE geeft het pad van het scriptbestand zelf. Dat is niet hetzelfde als $PWD, want $PWD is de huidige map waarin je staat.

#!/bin/bash
echo "De huidige folder is $PWD"
echo "Dit script bevindt zich in folder $BASH_SOURCE"

Waarom nuttig?

Als je script bestanden wil laden relatief ten opzichte van zichzelf, dan moet je vaak weten waar het script zelf staat, niet enkel waar jij op dat moment in de terminal staat.

source

Met source laad je een ander bestand in je huidige shell of script. Bash voert de inhoud dan uit alsof die rechtstreeks in je script stond.

Voorbeeld

source colors.inc

echo -e "${RED}rode tekst ${YELLOW}gele tekst ${RESET}"

Waarvoor gebruik je dit?

  • kleurcodes in apart bestand
  • gemeenschappelijke functies hergebruiken
  • configuratie apart houden

Alternatieve schrijfwijze

. colors.inc

De punt is gewoon een korte vorm van source.

Functies

Een functie is een herbruikbaar blok code. In plaats van dezelfde regels telkens opnieuw te schrijven, zet je ze in een functie en roep je die op wanneer nodig.

Basis syntax

function func1 {
  echo "This is an example of a function"
}

Functie aanroepen

func1

Volledig voorbeeld

#!/bin/bash

function func1 {
  echo "This is an example of a function"
}

count=1
while [ $count -le 5 ]
do
  func1
  count=$((count + 1))
done

echo "This is the end of the loop"
func1
echo "Now this is the end of the script"

Opmerkingen

  • Definieer een functie vóór je ze gebruikt
  • Gebruik unieke functienamen
  • Functienamen zijn hoofdlettergevoelig

Function exit status

De exit status van een functie is standaard de exit status van het laatste commando dat in die functie uitgevoerd werd.

Voorbeeld 1

func1() {
  echo "trying to display a non-existent file"
  ls -l badfile
}

echo "testing the function:"
func1
echo "The exit status is: $?"

Hier zal de exit status meestal 1 zijn, omdat ls -l badfile faalt en het laatste commando is.

Voorbeeld 2

func1() {
  ls -l badfile
  echo "This was a test of a bad command"
}

echo "testing the function:"
func1
echo "The exit status is: $?"

Hier wordt de exit status vaak 0, omdat echo het laatste commando is en dat meestal slaagt.

Zelf return gebruiken

check_bestand() {
  if [ -f "$1" ]; then
    return 0
  else
    return 1
  fi
}

check_bestand naam.txt
echo $?

Belangrijk: een function returnstatus moet tussen 0 en 255 liggen.

Function output

Een functie kan ook output teruggeven via echo. Die output kan je daarna opvangen in een variabele met command substitution.

dbl() {
  read -p "Enter a value: " value
  echo $((value * 2))
}

result=$(dbl)
echo "The new value is $result"

Onthouden

  • return is voor statuscodes
  • echo is voor echte outputwaarden

Function parameters

Net zoals een script kan een functie parameters krijgen. Binnen de functie gebruik je dan ook $1, $2, $#, enzovoort.

adden() {
  if [ $# -eq 0 ] || [ $# -gt 2 ]; then
    echo -1
  elif [ $# -eq 1 ]; then
    echo $(( $1 + $1 ))
  else
    echo $(( $1 + $2 ))
  fi
}

echo "Adding 10 and 15:"
value=$(adden 10 15)
echo $value

echo "Let's try adding just one number:"
value=$(adden 10)
echo $value

echo "Now trying adding no numbers:"
value=$(adden)
echo $value

Wat gebeurt hier?

  • $# = aantal parameters
  • 0 parameters of meer dan 2 = foutcode -1 als output
  • 1 parameter = tel die met zichzelf op
  • 2 parameters = tel beide op

Snelle snippets

IFS veilig aanpassen

OLDIFS=$IFS
IFS=$'\n'
# code
IFS=$OLDIFS

Snelle getopts basis

while getopts ':f:n' opt
do
  case "$opt" in
    f) echo "bestand: $OPTARG" ;;
    n) echo "droge run" ;;
    *) echo "ongeldige optie" ;;
  esac
done

Functie met returnstatus

bestaat_bestand() {
  [ -f "$1" ]
}

bestaat_bestand naam.txt && echo "OK" || echo "Niet gevonden"

Functie met output

verdubbel() {
  echo $(( $1 * 2 ))
}

result=$(verdubbel 21)
echo $result
