WhatIf en Confirm

WhatIf

-WhatIf toont wat een commando zou doen, zonder het echt uit te voeren.

Get-Process Code | Stop-Process -WhatIf

Confirm

-Confirm vraagt eerst bevestiging vooraleer een actie echt uitgevoerd wordt.

Get-Process Code | Stop-Process -Confirm

Dat wordt in les 5 expliciet als veilige manier van werken aangehaald. fileciteturn0file5L13-L14

Test-Connection

Test-Connection is in PowerShell het typische alternatief voor ping.

De les benadrukt dat ping nog bestaat, maar dat Test-Connection het PowerShell-alternatief is. fileciteturn0file5L15-L15

Alles is een object

Dit is één van de kernideeën van PowerShell.

  • Elk commando geeft objecten terug.
  • Die output wordt in het geheugen opgebouwd.
  • Via de pipeline stuur je objecten door naar volgende cmdlets.
  • Je kan die objecten daarna tonen, selecteren, sorteren of filteren.

Waarom is dat handig?

  • Je werkt niet met “rommelige tekst” maar met gestructureerde data.
  • Je kan eigenschappen rechtstreeks aanspreken.
  • Filteren en sorteren wordt veel eenvoudiger.

De slides leggen uit dat objecten data en methods bevatten, en dat de .NET-basis van PowerShell daarachter zit. fileciteturn0file5L17-L20

Get-Member

Get-Member laat zien met welk type object je werkt en welke properties of methods dat object heeft.

Get-Service | Get-Member

Belangrijk om te onthouden

  • Het type hangt af van het commando vóór Get-Member.
  • Sommige cmdlets veranderen het type van de output.
  • Als een commando geen output heeft, dan heeft Get-Member ook niets om te tonen.

De slides waarschuwen daar expliciet voor: Select-Object verandert bijvoorbeeld het objecttype en Get-Member toont vaak meer dan wat je standaard op het scherm ziet. fileciteturn0file5L23-L26

Sort-Object

Met Sort-Object sorteer je objecten op basis van één of meer properties.

Get-Process | Sort-Object Name
Get-Process | Sort-Object CPU -Descending
Get-Service | Sort-Object Status, Name

Belangrijk

  • Standaard sorteert PowerShell oplopend.
  • -Descending geeft aflopende sortering.
  • Je kan op meerdere properties sorteren.
  • Het objecttype verandert hierdoor niet.

Dat laatste staat heel duidelijk in de slides: sorteren verandert het objecttype niet. fileciteturn0file5L27-L28

Select-Object

Select-Object gebruik je om alleen die properties te tonen of door te geven die jij nodig hebt.

Get-Process | Select-Object Name, CPU
Get-Service | Select-Object Name, Status
Get-Process | Select-Object -First 5

Belangrijk

  • Je maakt vaak nieuwe, kleinere objecten voor output.
  • Daardoor verandert het type meestal naar een aangepast PSObject.
  • Gebruik Get-Member om dat verschil te zien.

Les 5 legt uit dat Select-Object nieuwe objecten maakt voor output. fileciteturn0file5L29-L31

Where-Object

Where-Object filtert objecten op basis van een zelf opgegeven voorwaarde.

Get-Service | Where-Object { $_.Status -eq "Running" }

Belangrijke ideeën

  • $_ verwijst naar het huidige object in de pipeline.
  • Met de puntoperator . spreek je properties aan.
  • Je gebruikt vergelijkingsoperatoren zoals -eq, -gt, -like en -match.

Filtervoorbeelden

Services die draaien

Get-Service | Where-Object { $_.Status -eq "Running" }

Complexer voorbeeld

Get-Service | Where-Object {
  $_.Status -eq "Running" -and $_.CanPauseAndContinue -eq $true
}

Tekst filteren met wildcard

Get-Service | Where-Object { $_.Name -like "win*" }

Regex filteren

Get-Service | Where-Object { $_.Name -match "^w" }

Het voorbeeld met Status -eq "Running" en CanPauseAndContinue -eq $true staat letterlijk in de slides. fileciteturn0file5L37-L38

Van terminal naar script

Les 5 schakelt daarna over naar scripting. Het basisidee:

  • Een script is een verzameling PowerShell-commando’s in een tekstbestand.
  • De extensie is .ps1.
  • Je kan extra controlestructuren toevoegen zoals variabelen, arrays, condities en lussen.
# voorbeeld.ps1
$name = Read-Host "Wat is jouw naam?"
Write-Output "Hallo $name"
